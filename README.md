# UAE CAIO Playbook

A planning workspace for UAE federal Chief AI Officers to govern the path to
**50% autonomous (agentic) AI government services by 2028**, per the directive of
HH Sheikh Mohammed bin Rashid Al Maktoum (23 April 2026) and the federal framework
approved by UAE Cabinet on 18 May 2026.

This repository uses **ArcKit v5.4.0** with the `arckit` core plugin and the
`arckit-uae` community overlay pre-enabled — a ready-to-use governance toolkit
that any federal entity can fork and adapt for its own agentic AI portfolio.

## The directive in brief

- **50% of UAE federal government services delivered by autonomous AI agents by 2028.**
- **Oversight:** HH Sheikh Mansour bin Zayed Al Nahyan, chair of the renamed
  Ministerial Council for AI and Development.
- **Execution coordinator:** Minister of Cabinet Affairs Mohammad Al Gergawi.
- **Performance metrics in the directive:** speed of adoption, implementation
  quality, workflow redesign.
- **Already live (May 2026):** four federal agents — Procurement, Tax Auditing,
  Customer Happiness, Technical Support; MoHRE autonomous work-permit
  decisions (from 1 May 2026); 80,000 employees in training.
- **Parallel Dubai mandate:** Sheikh Hamdan bin Mohammed has directed the
  emirate's private sector onto the same 24-month horizon.

## What this repo is for

A federal Chief AI Officer should be able to clone this repository and produce
a defensible, board-ready governance pack for their entity's agentic AI
portfolio — covering principles, stakeholders, requirements, business case,
roadmap, risk, transparency records, and UAE-specific compliance — in the
language of HH Sheikh Mansour's Ministerial Council, not generic IT-speak.

## Getting Started

### Prerequisites

- [Claude Code](https://claude.ai/code) installed
- GitHub Codespaces (recommended) or a local dev environment

### Setup

1. Open this repo in a GitHub Codespace, or clone locally.
2. Claude Code auto-installs via `.devcontainer/devcontainer.json`.
3. Both ArcKit plugins (`arckit` core, `arckit-uae` overlay) auto-enable via
   `.claude/settings.json`.
4. Restart Claude Code once to resolve the marketplace plugins.

## Recommended command sequence

A 24-month plan for a UAE federal CAIO running a full agentic-AI portfolio.
About 44 commands across 9 phases. Generated by `/arckit:start` for this
project profile (UAE federal, AI/ML portfolio, full lifecycle, quarters
timeline).

For a **minimum-viable pack** before your first Ministerial Council briefing,
stop after Phase 2 — you'll have principles, Charter compliance, stakeholders,
risk, business case, requirements, and autonomy-tier posture.

### Phase 1 — Foundation

```bash
/arckit:principles               # entity AI principles aligned to the Charter
/arckit-uae:uae-ai-charter       # 12-principle Charter compliance baseline
/arckit:stakeholders             # Ministerial Council, Cabinet Affairs, CAIO, ministry SROs
/arckit:risk                     # autonomy, contestability, oversight risks
```

The Charter sets the principle ceiling — run it right after `principles` so
every downstream artefact can point back to a Charter clause.

### Phase 2 — Business Justification

```bash
/arckit:sobc                          # Green Book 5-case, framed in directive KPIs
/arckit:requirements                  # candidate services for autonomous delivery
/arckit-uae:uae-ai-autonomy-tier      # Tier 1/2/3 posture per candidate service
```

Autonomy tier sits between requirements and design — it determines which
downstream controls (PDPL, IAS, ATRS depth) apply.

### Phase 3 — Data Design

```bash
/arckit:datascout                # training/operational data discovery
/arckit-uae:uae-classification   # Smart Data 5-tier (Open → Top Secret) per dataset
/arckit:data-model               # entities + relationships, classification-aware
/arckit-uae:uae-pdpl             # PDPL compliance (Federal Decree-Law 45/2021) — includes DPIA
/arckit-uae:uae-data-sharing     # cross-entity sharing agreements (collect-once)
/arckit-uae:uae-digital-records  # source-of-truth register, retention schedule
```

### Phase 4 — Strategy & Roadmap

```bash
/arckit:research                 # AWS/Azure/GCP options + UAE sovereign CSPs
/arckit:wardley                  # evolution map of the agent portfolio
/arckit:platform-design          # multi-sided platform across ministries
/arckit:strategy                 # executive synthesis
/arckit:roadmap                  # 24-month plan with directive KPIs
/arckit:diagram                  # C4 / DFD views
```

### Phase 5 — Procurement (Federal Decree-Law 11/2023)

```bash
/arckit-uae:uae-procurement      # ITT/RFP per MoF Digital Procurement Platform + ICV
/arckit:sow                      # statement of work per agent / per supplier
/arckit:evaluate                 # evaluation framework + ICV scoring
```

### Phase 6 — Identity, Hosting, Cyber

```bash
/arckit-uae:uae-uaepass          # identity integration (OIDC, Basic vs Verified)
/arckit-uae:uae-cloud-residency  # sovereign cloud per classification
/arckit-uae:uae-ias              # NESA-aligned IAS (188 controls, P1–P4)
/arckit:hld-review               # validate HLD against principles + requirements
/arckit:dld-review               # validate DLD per agent
/arckit:adr                      # one ADR per material decision (autonomy, hosting, identity, model)
```

### Phase 7 — Build & Operate

```bash
/arckit:backlog                  # backlog from requirements + design
/arckit:devops                   # CI/CD, IaC
/arckit:mlops                    # model lifecycle, drift detection (AI/ML modifier)
/arckit:operationalize           # runbooks, on-call, DR/BCP
/arckit:finops                   # cloud cost management
```

### Phase 8 — Responsible AI & Assurance (per agent)

```bash
/arckit:ai-playbook                       # responsible AI assessment
/arckit:atrs                              # ATRS — one per live or planned agent (≥4 today)
/arckit-uae:uae-priorities-alignment      # NIS 2031 / AI 2031 / We the UAE 2031
/arckit-uae:uae-zero-bureaucracy          # service-catalogue review per UAE Code
```

### Phase 9 — Conformance & Board Reporting

```bash
/arckit:traceability             # end-to-end matrix
/arckit:principles-compliance    # principles adherence scorecard
/arckit:conformance              # ADR + drift + cross-decision consistency
/arckit:analyze                  # deep governance analysis
/arckit:story                    # narrative for the Ministerial Council
/arckit:presentation             # MARP slide deck for board briefings
/arckit:pages                    # publishable governance site
```

### Accelerators

- **`/arckit:build`** — once Phases 1–2 are done, dispatches many later
  commands in parallel waves with resumable state in `.arckit/state.json`.
  Useful for Phase 3, 6, 8 fan-out.
- **`/arckit:navigator`** — coverage gaps and stale artefacts.
- **`/arckit:health`** — pre-board-meeting check for forgotten ADRs and
  overdue reviews.

### Suggested cadence

- **Month 1**: Phases 1–2 — first Ministerial Council artefact set
- **Month 2–3**: Phases 3–4 — board-ready strategic pack
- **Month 4–5**: Phases 5–6 — supplier engagement and assurance design
- **Month 6+**: Phases 7–9 rolling; re-run Phase 8 ATRS per new agent going live
