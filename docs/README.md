# UAE CAIO Playbook — Documentation

This project uses **ArcKit v5.4.0** for enterprise architecture governance,
with the `arckit-uae` community overlay enabled for UAE-specific compliance.

## Purpose

Help UAE federal Chief AI Officers plan their entity's path to **50% autonomous
AI government services by 2028**, per the 23 April 2026 directive of
HH Sheikh Mohammed bin Rashid Al Maktoum (federal framework approved by
UAE Cabinet on 18 May 2026, overseen by HH Sheikh Mansour bin Zayed's
Ministerial Council for AI and Development).

## Getting Started

Run ArcKit commands using the Claude Code plugin. See the project root
[README](../README.md) for the recommended first-command sequence.

Headline commands for the agentic AI mandate:

- `/arckit.principles` — define architecture principles
- `/arckit.uae-charter-ai` — UAE AI Charter compliance
- `/arckit.stakeholders` — Ministerial Council, Cabinet Affairs, CAIO governance
- `/arckit.requirements` — candidate services for autonomous delivery
- `/arckit.sobc` — Strategic Outline Business Case
- `/arckit.roadmap` — 24-month phased plan
- `/arckit.risk` — agent autonomy, oversight, contestability
- `/arckit.atrs` — Algorithmic Transparency Records (per agent)
- `/arckit.ai-playbook` — Responsible AI deployment
- `/arckit.uae-pdpl` — UAE PDPL
- `/arckit.uae-cybersecurity` — UAE/NESA cyber controls

See [the ArcKit Documentation](https://tractorjuice.github.io/arc-kit/) for
the full command reference (71 core + 12 UAE overlay).
